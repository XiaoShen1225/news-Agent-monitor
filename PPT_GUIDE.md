### 第 4 页：FetcherAgent — 网页抓取与变更检测

**技术方案**：
- 静态抓取 (httpx) + 动态渲染 (Playwright 无头 Chromium)
- 6 步渐进滚动 + 底部滚动 + 回滚，触发懒加载
- HTML 从 79KB → 133KB，提取条目从 54 → 241

**变更检测**：
- 剥离 script/style/noscript 标签 + 空白 → SHA256
- 哈希相同 → 直接跳过下游 Agent（**零 Token 消耗**）

**关键数字**：241 条单次抓取 | ~20 秒处理时间 | 零 API 费用

---

### 第 5 页：ParserAgent — 结构化提取（零 LLM Token）

**核心设计决策**：为什么不用 LLM？
- 百度新闻按板块组织，DOM 结构已含分类信息
- LLM 引入延迟、成本、不确定性 → 选择纯 DOM 遍历

**实现方案**（3 步）：
1. 按文档顺序遍历 `soup.descendants`
2. 遇到章节关键词 → 更新 `current_tag`（长关键词优先匹配）
3. 遇到 `<a>` 标签 → 提取 title + href + current_tag

**过滤机制**：
- 标题长度 6-200 字符 | 20+ 噪声正则 | 15+ UI 标签黑名单
- 链接文本保护（新闻标题不会被误判为章节标记）

**结果**：241 条新闻 × 11 个分类，**0 Token 消耗**

---

### 第 6 页：AnalyzerAgent — 变更分析

**Diff 算法**（标题为主键）：
```
new_items     = curr_titles - prev_titles    → 新增
removed_items = prev_titles - curr_titles    → 移除
modified_items = 同标题但 tag/summary 变化    → 修改
```

**趋势计算**：
- 最近 3 次快照均值 vs 更早快照均值
- `> 1.1×` → 上升 | `< 0.9×` → 下降 | 其他 → 稳定

**设计亮点**：哈希检测与标题 Diff 两级过滤，避免误报

### 第 8 页：数据存储与查询

**三层存储架构**：

| 层级 | 格式 | 用途 |

**CLI 查询命令**：

### 第 9 页：自进化机制 (Evolution)

**记忆系统**：
- 每次运行记录：时间戳、站点、提取数、变更数、置信度、耗时
- 持久化到 `evolution/memory.json`

**两个优化方向**：
1. **调度调优**：连续 3 次无变化 → 间隔翻倍（省资源）；频繁变化 → 间隔缩短（及时捕获）
2. **提示词进化**：当 LLM 置信度 < 0.5，自动追加格式强化指令

## 三、PPT 制作建议

### 风格
- 深色背景 + 白色/青色文字，技术感
- 每页控制在 3-5 个要点，不要放整段文字
- 代码用等宽字体高亮

### 配图来源
- 系统运行后在 `outputs/charts/today/` 找到 PNG 图表，直接插入 PPT
- 架构图用 draw.io 或 PPT 自带形状重绘（不要放 ASCII 图）

### 重点强调页
- 第 5 页（零 Token 解析）和第 7 页（10 种图表）是加分项
- 第 10 页实验数据截图最有说服力

### 演说时间控制
- 12 页 PPT，建议控制在 8-10 分钟
- 架构 + Agent 设计（第 3-7 页）占一半时间
- 实验数据（第 10 页）2 分钟，留时间给 Q&A
